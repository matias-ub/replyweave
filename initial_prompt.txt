# AUTHENTICATION

There is no login or signup flow. Authentication is intentionally out of scope.

All write endpoints must require a header:

```
X-User-Id
```

This must contain a valid UUID.

The UUID is treated as the acting user without verification.

If the header is missing or invalid, return HTTP 400.

Do not implement:

* JWT
* OAuth
* sessions
* token systems

# DATABASE DESIGN

Use PostgreSQL extension:

```
pgcrypto
```

UUIDs must be generated in the database using:

```
gen_random_uuid()
```

---

## Users table

```
users
  id            uuid primary key default gen_random_uuid()
  username      unique
  created_at    timestamp
```

---

## Posts table

```
posts
  id                    uuid primary key default gen_random_uuid()
  title                 text
  prompt                text
  conversation_summary  text
  conversation_json     jsonb

  source_platform       text
  source_model          text

  remix_of              uuid nullable

  created_by            uuid references users(id)

  created_at            timestamp

  likes_count           integer default 0
  comments_count        integer default 0
  remix_count           integer default 0

  message_count         integer
  token_estimate        integer
  language              text

  embedding             vector(384)
```

Embeddings must be generated from:

```
conversation_summary
```

---

# CONVERSATION FORMAT

Canonical conversation format:

```json
{
  "messages": [
    {
      "id": "msg1",
      "role": "user",
      "content": [
        {"type": "text", "text": "Explain recursion like I'm 10"}
      ]
    },
    {
      "id": "msg2",
      "role": "assistant",
      "content": [
        {"type": "text", "text": "Imagine a box that contains another box"}
      ]
    }
  ]
}
```

Store this structure in:

```
posts.conversation_json
```

Messages must also be parsed and inserted into the `messages` table.

# EMBEDDINGS

Use sentence-transformers model:

```
all-MiniLM-L6-v2
```

Embeddings must be generated from:

```
conversation_summary
```

Embedding generation must:

* run in a FastAPI `BackgroundTask`
* never block the request

When a post is created:

```
embedding = NULL
```

A background task must compute and persist it later.

The embedding model must be loaded **once at application startup** and reused across requests.

## Mock Implementation Requirement

ChatGPT and Claude share pages render content via JavaScript.

The HTML returned by `requests` contains only a shell.

Therefore implement a **mock importer**.

For URLs matching:

```
chatgpt.com/share/*
claude.ai/share/*
```

Return a **hardcoded conversation fixture**.

Add a TODO comment explaining that production would require a headless browser such as Playwright to extract `__NEXT_DATA__` or equivalent JS payload.

---

## Import Pipeline

Steps:

1 Detect platform from URL
2 Return mock fixture
3 Normalize conversation
4 Extract prompt from first user message
5 Generate conversation_summary (first user message + first assistant response)
6 Schedule embedding background task
7 Return preview post

# API ENDPOINTS

```
POST   /users
Create a user with a username. No auth required. Returns the created user including their UUID

POST   /posts
Create post (requires X-User-Id)

POST   /import
Import conversation from share link (requires X-User-Id)

GET    /posts
Return latest posts ordered by created_at DESC with cursor pagination

GET    /posts/{post_id}
Return a single post

GET    /posts/{post_id}/messages
Return a list of the parsed messages from a post

GET    /users/{user_id}
Return user metadata

GET    /users/{user_id}/posts
Return posts created by the user

GET    /search/posts?q=<query>
Perform semantic search using pgvector

GET    /posts/{post_id}/similar
Return similar posts using embedding similarity

POST   /posts/{post_id}/like
(requires X-User-Id)

POST   /posts/{post_id}/remix
Creates a new post derived from an existing conversation. The new post references the original via remix_of and may extend or modify the conversation (requires X-User-Id)
```

# CURSOR PAGINATION

Use a **compound cursor**.

Cursor format:

```
base64(created_at|post_id)
```

Example decoded cursor:

```
2026-03-14T10:15:30Z|uuid
```

Query pattern:

```
WHERE (created_at, id) < (:created_at, :id)
ORDER BY created_at DESC, id DESC
LIMIT :limit
```

Response format:

```json
{
  "data": [...],
  "next_cursor": "base64_cursor_or_null"
}
```

---

# SEMANTIC SEARCH

Steps:

1 Generate query embedding
2 Perform pgvector similarity search

Example SQL:

```
SELECT *
FROM posts
WHERE embedding IS NOT NULL
ORDER BY embedding <-> query_embedding
LIMIT 20
```
